# 撰寫守則

## 1. 章節字數硬規則
1. 每章正文最低 `3000` 字，低於 3000 不交稿。
2. 建議上限 `3600` 字，避免拖節奏。
3. 字數計算以正文為主，不含章標題、作者註、備註清單。
4. 每章至少要有一個小高潮或明確反轉。

## 2. 每章固定結構（3000字模板）
1. 開場鉤子（300-500字）：立即給危機、懸念或收益目標。
2. 探索推進（900-1100字）：找線索、試能力、擴情報。
3. 衝突爆發（900-1100字）：戰鬥、交易翻車、勢力壓迫三選一或混合。
4. 收益回收（400-600字）：主角拿到實際成長，例：新模板、新據點、新材料。
5. 章尾鉤子（150-300字）：下一章危機或更大利益入口。

## 3. 主角爽感規則
1. 每章至少一次「能力有效回報」，不能只有吃虧。
2. 吃虧章必須同章埋反打手段，最晚兩章內反殺或回收。
3. 爽點優先順序：修復出寶物 > 鼠群戰術碾壓 > 權謀反制 > 境界突破。
4. 金手指要強，但每次強力使用都要付代價（負荷、污染、材料消耗）。

## 4. Markdown 寫作風格規則

### 4.1 章節標題格式（固定）
```md
# 第X章 章名
```

### 4.2 場景切換（固定）
```md
---
```

### 4.3 系統/金手指提示（固定）
```md
> 【鍊成爐提示】缺件：相位線圈 x1（來源：拋投塔-3區）
> 【魂譜提示】當前污染值：21%（超過30%將觸發失控）
```

### 4.4 爽點句加重（適量）
- 關鍵一句可用 `**粗體**` 強化，但每章建議不超過 8 句。
- 禁止整段粗體，避免閱讀疲勞。

### 4.5 收束資訊（融入正文，不另起標題）
```md
（收束段落示例）
李塵把鍊成爐回報和井口驗簽記錄對在一起，確認骨鑰補完已到100%。
這次真正拿到的是六小時演訓窗與可續用的5%試行配額。
代價也很直接：裂痕回流再上跳一格，必須強制休整兩天。
他把路線圖往下推一格，決定下階段先做啟取試刀，不再硬壓全額度。
```

## 5. 對話與敘事比例
1. 建議比例：敘事 70%、對話 30%。
2. 戰鬥章可調整為敘事 60%、對話 40%。
3. 連續兩頁都在解說設定時，必須插入行動或衝突。

## 6. 角色出場規則（前30章）
1. 主角每章視角占比至少 80%。
2. 配角每章最多重點描寫 2 人，避免資訊過載。
3. 反派每次出場都要推進威脅，不做無效露臉。

## 7. 伏筆與回收規則
1. 每章至少埋 1 個小伏筆（物件、台詞、地點異常都可）。
2. 小伏筆 3-5 章內回收；大伏筆一卷內回收。
3. 卷尾必須回收本卷核心承諾：第一卷是「拿到第一枚神格碎片」。

## 8. 交稿前自檢清單（每章）
- 是否超過 3000 字。
- 是否出現明確衝突與回報。
- 是否有章尾鉤子。
- 金手指是否有代價，不是無腦碾壓。
- 世界設定是否透過劇情呈現，不是純說明文。
- 是否與既有章節邏輯一致（道具、傷勢、資源數量有延續）。

## 9. 建議檔案模板（可直接複製）
```md
# 第X章 章名

（開場鉤子）

---

（探索推進）

> 【鍊成爐提示】...

---

（衝突爆發）

---

（收益回收）

（收束段：把查驗結果、收穫、代價、下階段方向寫進敘事）

---

（章尾鉤子）
```

## 10. 防第四面牆準則（絕對禁令）
1. **禁用元數據語彙**：正文（Prose）絕對禁用到「第X卷」、「第X章」、「ChXX」、「Chapter XX」、「卷末」、「本章」等。
    - *錯誤範例*：「在第一卷中他曾見過這東西。」
    - *正確範例*：「在最初落入廢星的那段日子裡，他曾見過這東西。」
2. **禁用編輯標記**：正文禁用「補寫段落」、「作者註」、「待補」等。
3. **撤銷章末宣告**：除非是特殊的預告段落（且被 > 引用），否則正文末尾不得出現「第X卷完」等宣告。
4. **時空座標轉換**：
    - 若要回溯舊劇情，請使用「劇情里程碑」：如「九號井戰役之後」、「在 T-17 據點剛建立時」、「與蘇芮初次連結時」。
    - 禁使用「三章前」、「Ch45 那次」。
5. **情境化資訊**：任何設定說明必須有「載體」。
    - 禁：「如前文所述，神格碎片的融合需要時間。」
    - 坤：「終端屏幕上跳出的進度條提醒著李塵，神格碎片的融合仍處於膠著狀態。」
6. **對話去元化**：角色對話中不得出現任何暗示自己是小說角色的詞彙。
7. **嚴格自檢**：交稿前必須 grep 檢查「卷」、「章」、「Ch」關鍵字。
