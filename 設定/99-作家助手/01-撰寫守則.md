# 撰寫守則

## 1. 章節字數硬規則
1. 每章正文最低 `3000` 字，低於 3000 不交稿。
2. 建議上限 `3600` 字，避免拖節奏。
3. 字數計算以正文為主，不含章標題、作者註、備註清單。
4. 每章至少要有一個小高潮或明確反轉。

## 2. 每章固定結構（3000字模板）
1. 開場鉤子（300-500字）：立即給危機、懸念或收益目標。
2. 探索推進（900-1100字）：找線索、試能力、擴情報。
3. 衝突爆發（900-1100字）：戰鬥、交易翻車、勢力壓迫三選一或混合。
4. 收益回收（400-600字）：主角拿到實際成長，例：新模板、新據點、新材料。
5. 章尾鉤子（150-300字）：下一章危機或更大利益入口。

## 3. 主角爽感規則
1. 每章至少一次「能力有效回報」，不能只有吃虧。
2. 吃虧章必須同章埋反打手段，最晚兩章內反殺或回收。
3. 爽點優先順序：修復出寶物 > 鼠群戰術碾壓 > 權謀反制 > 境界突破。
4. 金手指要強，但每次強力使用都要付代價（負荷、污染、材料消耗）。

## 4. Markdown 寫作風格規則

### 4.1 章節標題格式（固定）
```md
# 第X章 章名
```

### 4.2 場景切換（固定）
```md
---
```

### 4.3 系統/金手指提示（固定）
```md
> 【鍊成爐提示】缺件：相位線圈 x1（來源：拋投塔-3區）
> 【魂譜提示】當前污染值：21%（超過30%將觸發失控）
```

### 4.4 爽點句加重（適量）
- 關鍵一句可用 `**粗體**` 強化，但每章建議不超過 8 句。
- 禁止整段粗體，避免閱讀疲勞。

### 4.5 清單式資訊（用在修復/交易/戰利品）
```md
## 本章戰利品
- 神性粉末 x3
- 完整濾能管 x1
- 可修復掃描器核心 x1
```

## 5. 對話與敘事比例
1. 建議比例：敘事 70%、對話 30%。
2. 戰鬥章可調整為敘事 60%、對話 40%。
3. 連續兩頁都在解說設定時，必須插入行動或衝突。

## 6. 角色出場規則（前30章）
1. 主角每章視角占比至少 80%。
2. 配角每章最多重點描寫 2 人，避免資訊過載。
3. 反派每次出場都要推進威脅，不做無效露臉。

## 7. 伏筆與回收規則
1. 每章至少埋 1 個小伏筆（物件、台詞、地點異常都可）。
2. 小伏筆 3-5 章內回收；大伏筆一卷內回收。
3. 卷尾必須回收本卷核心承諾：第一卷是「拿到第一枚神格碎片」。

## 8. 交稿前自檢清單（每章）
- 是否超過 3000 字。
- 是否出現明確衝突與回報。
- 是否有章尾鉤子。
- 金手指是否有代價，不是無腦碾壓。
- 世界設定是否透過劇情呈現，不是純說明文。
- 是否與既有章節邏輯一致（道具、傷勢、資源數量有延續）。

## 9. 建議檔案模板（可直接複製）
```md

（開場鉤子）

---

（探索推進）

> 【鍊成爐提示】...

---

（衝突爆發）

---

（收益回收）

## 本章戰利品
- ...

---

（章尾鉤子）
```
